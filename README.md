# portfolio

純前端的個人投資組合追蹤工具：跨券商持倉彙總、XIRR 年化報酬率、股票質押維持率監控與壓力測試。

## 資料安全模型（重要）

**本站不收集、不傳輸、不儲存任何使用者資料。** 所有輸入（持倉、借款、記帳現金流）都只存在**你自己瀏覽器的 IndexedDB**：

- 網站是純靜態頁面（GitHub Pages），沒有後端、沒有資料庫
- 「匯入 CSV」由瀏覽器本機讀取解析，檔案不會離開你的裝置
- 對外連線僅限三個唯讀報價 API（TWSE／Twelve Data／open.er-api.com），由 CSP `connect-src` 白名單強制——即使發生 XSS 或供應鏈攻擊，資料也沒有可外送的通道
- 每個裝置/瀏覽器的資料各自獨立；清除瀏覽器網站資料會一併刪除，請定期使用 JSON 備份

## 使用教學

線上版：**https://guitarbien.github.io/portfolio-app/**（所有資料只存在你的瀏覽器，見上節）

### 1. 建立帳戶與持倉（「持倉」頁）

1. 先建券商帳戶：帳戶名稱、券商、幣別（台股 TWD／美股 USD）、目前現金餘額 → **新增帳戶**
2. 再建「開帳快照」：選帳戶、輸入代號（如 `0050`、`VOO`）、名稱、市場、槓桿倍數（一般股票填 `1`，正2 ETF 填 `2`）、股數 → **新增持倉**

> 開帳快照是**一次性的起始基準**，每檔標的只建一次；之後的買賣請記在「紀錄」頁的交易，系統會自動合併計算目前持倉。

### 2. 登錄借款（「借款」頁）

- **股票質押**：借款餘額、年利率、追繳門檻（預設 130%，依你的合約調整，如元大證金 140%）、回補標準（預設 166%）、擔保品代號與股數
- **理財型房貸**：核定額度、目前動用餘額、年利率（無維持率，未動用額度會計入壓力測試的「補繳子彈」）

### 3. 報價

- 台股收盤價與美元匯率：開啟 App 時自動抓取（每日一次）
- 美股：到「設定」頁填入 [Twelve Data](https://twelvedata.com/) 免費 API key
- 任何標的都可在「設定」頁**手動輸入收盤價**（手動值永遠優先於自動抓取）

### 4. 看儀表板

- **KPI 卡**：淨值、總曝險、槓桿倍率（總曝險÷淨值，槓桿 ETF 以市值×倍數計入）、年化報酬率 XIRR
- **質押卡**：每筆借款的維持率（綠＝安全、黃＝接近預警、紅＝低於追繳門檻）與「還能跌 X%」
- **壓力測試**：拉動「大盤跌幅」滑桿，即時看每筆質押的需補擔保金額／或還款金額，以及補繳子彈（現金＋房貸未動用額度）夠不夠

### 5. 匯入歷史記帳算長期報酬（「紀錄」頁底部）

1. 選匯入帳戶 → 上傳記帳 CSV（支援民國年、千分位）
2. 對映欄位：哪欄是日期、哪欄是金額、哪欄是類別
3. 為每個類別值指定意義：投入／股利／提領／忽略（沒有類別欄時，以金額正負判定：正＝投入、負＝提領）
4. 預覽（無法解析的列會列出行號與原因）→ 匯入 → 儀表板出現**年化報酬率（XIRR）**與起算日

只需要「日期＋金額」的現金流記錄就能算出正確的年化報酬率，不需要歷史成交明細。

### 6. 日常維護

- 買賣股票 → 「紀錄」頁新增交易（股數買正賣負）
- 領到股利、出入金 → 「紀錄」頁新增現金流
- 現金餘額、借款餘額變動 → 「持倉」／「借款」頁各項目旁直接編輯儲存

### 注意事項

- 資料存在瀏覽器 IndexedDB：**清除瀏覽資料會一併刪除**，跨裝置不互通（JSON 備份/還原功能開發中）
- 維持率以每日收盤價計算（與券商計算基準一致），非盤中即時

## 開發

```bash
npm ci
npm run dev        # 開發伺服器
npm run test       # 測試
npm run coverage   # 測試＋覆蓋率門檻（85%）
npm run build      # 產出 dist/（含 CSP meta）
```

推送到 `main` 會自動執行測試閘門並部署到 GitHub Pages（`.github/workflows/deploy.yml`）。

## 技術

React 19 + TypeScript + Vite、Dexie（IndexedDB）、Vitest + Testing Library。設計文件見 `docs/superpowers/specs/`。
